# GGM 개발 연습 폴더

이 폴더는 개발 연습용 프로젝트입니다.

## 기술 스택 (사용하는 도구들)

- **Next.js** — 웹사이트/앱을 만드는 프레임워크 (도구 모음). 페이지, 화면, 기능을 구성하는 뼈대 역할을 합니다.
- **Supabase** — 인터넷에 있는 데이터베이스 (정보 저장소). 회원정보, 게시글, 거래내역 등 모든 데이터를 여기에 저장합니다.
- **Vercel** — 완성된 웹사이트를 인터넷에 올려서 누구나 접속할 수 있게 해주는 배포 서비스입니다.

## 현재 프로젝트

- **가계부 앱** (`index.html`) — 팝아트 스타일의 수입/지출 관리 앱. Supabase와 연결되어 데이터가 클라우드에 저장됩니다.

## Claude의 작업 방식

### 진행 상황 안내
- 작업 중에 **지금 무엇을 하고 있는지** 자주 알려드립니다.
- 단계가 여러 개일 때는 "1단계 완료 → 2단계 진행 중" 식으로 표시합니다.

### 작업 범위
- **사용자가 지시한 것만** 단계적으로 수행합니다. 요청하지 않은 추가 기능이나 변경은 하지 않습니다.
- 할 수 있는 작업은 최대한 직접 처리합니다.
- **사람이 직접 해야 하는 것** (예: 브라우저 로그인, 환경 변수 입력, 결제 등)만 쉬운 말로 안내합니다.

### 작업 완료 후
- 개발이 끝나면 **결과를 바로 눈으로 확인할 수 있도록** 미리보기를 열어드립니다.
- 배포(Vercel에 올리기)가 완료되면 실제 주소를 알려드립니다.

## 용어 설명 (모르는 말이 나오면 여기서 확인하세요)

| 용어 | 쉬운 설명 |
|------|-----------|
| 배포 (Deploy) | 만든 웹사이트를 인터넷에 올려서 다른 사람들도 볼 수 있게 하는 것 |
| 커밋 (Commit) | 작업 내용을 저장하는 것. 마치 "저장" 버튼처럼 변경사항을 기록합니다 |
| 브랜치 (Branch) | 원본을 건드리지 않고 별도로 작업하는 복사본 공간 |
| 환경변수 (ENV) | 비밀번호처럼 외부에 노출되면 안 되는 설정값 (API 키 등) |
| 마이그레이션 (Migration) | 데이터베이스 구조를 바꾸는 작업 |
| 컴포넌트 (Component) | 재사용 가능한 화면 조각 (버튼, 카드, 메뉴 등) |
| API | 앱끼리 정보를 주고받는 통로 |
| 빌드 (Build) | 코드를 실제로 작동하는 웹사이트 파일로 변환하는 과정 |
| 로컬 (Local) | 내 컴퓨터에서만 볼 수 있는 상태 (아직 인터넷에 올리지 않은 것) |

## 사람이 직접 해야 하는 작업 (Claude가 대신할 수 없는 것들)

Claude가 이런 상황에서는 **명확하게 안내 메시지**를 드립니다:
- Supabase 대시보드에서 직접 설정 변경
- Vercel 환경변수 입력
- GitHub/Vercel/Supabase 로그인
- 도메인 구매 및 연결
- 결제 관련 작업
